사설 ip : 어떤 네트워크 안에서만 내부적으로 사용되는 고유한 주소
공인 ip : ISP 가입을 통해 제공받은 IP
ISP(Internet Service Provider의 약자로 KT, LG, SKT와 같이 인터넷을 제공하는 통신업체)

네트워크 접근성: 사설 IP 주소는 인터넷에서 직접 접근할 수 없습니다. 사설 IP 주소는 일반적으로 내부 네트워크에서만 사용되며, 인터넷 상에서 유니크한 공인 IP 주소로 변환되어야만 외부에서 접근할 수 있습니다.

공인 IP 주소 부족: IPv4 주소 체계는 공인 IP 주소가 한정되어 있고, 이 주소들은 인터넷 서비스 제공자(ISP)에 의해 할당됩니다. 현재 많은 기기들이 인터넷에 접속하는데 필요한 IP 주소가 계속 증가함에 따라 공인 IP 주소 부족 문제가 발생하고 있습니다.

보안 위험: 사설 IP 주소로 직접 웹 서버를 운영하는 경우, 보안상의 위험이 증가할 수 있습니다. 외부에서 직접 접근할 수 없기 때문에 일반적으로 방화벽이나 라우터의 포트 포워딩 설정을 통해 사설 IP 주소를 공인 IP 주소와 연결해야 하며, 이로 인해 보안적인 취약점이 발생할 수 있습니다.

DNS 문제: 사설 IP 주소는 일반적으로 도메인 이름과 연결되지 않습니다. 웹 서버를 도메인 이름으로 접근하려면 도메인 이름과 공인 IP 주소를 연결하는 DNS(Domain Name System) 레코드가 필요합니다. 사설 IP 주소로는 이러한 DNS 설정을 할 수 없기 때문에 외부에서 도메인 이름으로 서버에 접근하는 것이 어렵습니다.

DHCP(동적 호스트 구성 프로토콜)
장치(예: 컴퓨터, 스마트폰 및 기타 네트워크 지원 장치)가 필요한 네트워크 구성 정보를 자동으로 얻을 수 있도록 하는 네트워크 프로토콜
IP가 포함된다.

Port Fowarding
포트 포워딩은 라우터의 공용 IP 주소에 있는 하나의 외부 포트에서 로컬 네트워크에 있는 특정 장치의 내부 IP 주소 및 포트로 트래픽을 전달하기 위해 NAT(네트워크 주소 변환)에 사용되는 기술

사설IP로도 절차만 밟으면 가능하기 때문에

가능함.

쪼개지는 이유 : 공유기에 NAT(네트워크 주소 변환)기능이 있어 공유기가 내부 네트워크의 여러 기기들이 하나의
공인 IP 주소를 공유하게 된다.


허브 : L2스위치로 허브에 연결된 한 디바이스에서 발생하는 Collision이 허브에 연결된 모든 영역에 노드들에게 영향을 미쳐 통신을 하지 못한다.
자신에게 연결된 디바이스들의 MAC 주소와 포트가 기록된 MAC 주소 테이블을 가지고 있다.
따라서 프레임이 자신에게 오면 그 것의 목적지가 어디인지 파악하여 그 디바이스에게 프레임을 보내기 때문에 트래픽이 훨씬 효율적이다.
